# 吃什麼？APP（WhatToEat）

**版本：** V1.0（MVP 初版）  
**文件撰寫日：** 2025/09/19  
**專案經理：** 原愷

---

## 一、專案簡介

吃什麼？APP 旨在解決日常生活中「不知道要吃什麼」的選擇困難問題。平台結合美食分享、社群互動與點數機制，讓用戶每日透過上傳照片與心得解鎖靈感，商家則可參與互動並透過點數提升曝光，打造一個具備良性循環的輕社交型美食生態。

---

## 二、產品定位與目標

### 產品定位

一款透過每日動態清除與熱門貼文留存設計，結合點數經濟與商家參與的社群型美食分享 App。

### 目標用戶

- 18～45 歲有拍照習慣的美食愛好者
- 尋求餐廳靈感的上班族／學生
- 想要曝光餐點或活動的中小型店家

### MVP 目標

- 完成核心功能開發並提供封閉測試版本
- 初步驗證用戶互動行為與商家點數系統可行性

## 三、功能模組與需求說明

### （A）一般使用者模組

1. **註冊／登入功能**：支援 Email、Google、Facebook、Apple ID。
2. **發文機制**：
   - 上傳 1 張照片（壓縮處理）。
   - 填寫心得（限 150 字）。
   - 地點可手動輸入或自動定位建議。
   - 每日最多發文 3 篇。
3. **貼文壽命機制**：
   - 預設壽命 24 小時。
   - 超過即自動清除（除非進入熱門或被商家延長）。
4. **點數獲取邏輯**：
   - 發文得 1 點。
   - 每日最多 3 點。
   - 被商家延長／進入熱門可再獲額外點數。
5. **商城頁面**：使用點數兌換虛擬商品或折價券（初期為展示用途）。
6. **熱門區展示邏輯**：
   - 被按讚數 ≥ 50 自動進入熱門區。
   - 熱門貼文不會被每日排程清除。

### （B）商家模組

1. **商家帳號註冊／登入**。
2. **商家貼文後台**：查看、編輯、自家被標記的貼文紀錄。
3. **點數購買功能**：以金額購買點數（綁定後續商店延伸）。
4. **推廣功能**：
   - 選定貼文延長壽命（每 24 小時需消耗點數）。
   - 或購買熱門曝光位，獲得輪播展示。
5. **回饋用戶點數**：商家延長某貼文，可選擇回饋創作者點數（強化互動）。

### （C）系統後台模組

1. **每日自動清除機制**：排程排除熱門貼文與商家延長之內容。
2. **圖片壓縮與分類儲存**：採用 AWS S3 或 Firebase。
3. **點數流通紀錄後台**：含用戶、商家點數異動日誌。
4. **用戶／商家權限控管與資料維護**。
5. **防濫用與安全設計**：驗證機制、圖片審查 API 預留。

## 四、技術建議與架構初稿

| 類別   | 建議技術方案 |
| ------ | -------------- |
| 前端   | Flutter（跨平台優先） |
| 後端   | Node.js + Express／Firebase Function（MVP 階段快速開發） |
| 資料庫 | Firestore（NoSQL） |
| 身分驗證 | Firebase Auth |
| 圖片儲存 | Firebase Storage／AWS S3 |
| 推播通知 | Firebase Cloud Messaging |

---

## 五、UI/UX 初版建議頁面（共 6 頁）

1. 首頁：隨機推薦＋熱門貼文。
2. 發文頁：圖片上傳＋文字填寫＋地點標籤。
3. 我的頁面：個人貼文／點數查詢。
4. 商城頁面：可兌換內容展示。
5. 商家後台：貼文管理、點數延長紀錄。
6. 通知中心：系統通知、點數異動、被標記提醒。

## 七、附註

- MVP 僅支援圖片，不支援影片、留言與聊天室。
- 未開放貼文搜尋與分類篩選（後期版本支援）。
- 商城僅展示點數用途，不涉及金流與兌獎（待後續串接）。
